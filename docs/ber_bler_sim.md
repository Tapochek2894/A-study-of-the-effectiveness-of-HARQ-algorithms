# BER/BLER симулятор для кодов Хэмминга

Утилита `ber_bler_sim` моделирует код Хэмминга в двоичном симметричном канале (BSC)
и считает BER/BLER для заданных вероятностей ошибки на бит.

## Сборка

```bash
cmake -S . -B build
cmake --build build --target ber_bler_sim
```

## Запуск

```bash
./build/ber_bler_sim --r 3 --blocks 20000 --p 0.001,0.005,0.01
```

Пример диапазона (логарифмическая сетка по умолчанию):
```bash
./build/ber_bler_sim --r 3 --blocks 200000 --p-start 1e-4 --p-end 1e-2 --p-points 9
```

Линейная сетка:
```bash
./build/ber_bler_sim --r 3 --blocks 200000 --p-start 0.001 --p-end 0.01 --p-points 10 --p-linear
```

Параметры:
- `--r <parity_bits>`: число проверочных битов `r` (код `(2^r-1, 2^r-1-r)`), по умолчанию `3`.
- `--blocks <count>`: число переданных блоков, по умолчанию `10000`.
- `--seed <seed>`: seed для генератора случайных чисел, по умолчанию `5489`.
- `--p <p1,p2,...>`: список вероятностей ошибки бита в BSC, по умолчанию
  `1e-4,5e-4,1e-3,5e-3,1e-2`.
- `--p-start <value>`: начало диапазона `p` (используется вместе с `--p-end`).
- `--p-end <value>`: конец диапазона `p` (используется вместе с `--p-start`).
- `--p-points <n>`: число точек в диапазоне `p` (минимум 2).
- `--p-linear`: использовать линейную сетку (по умолчанию логарифмическая).

## Формат вывода (CSV)

Заголовок:
```
p,ber,bler,bit_errors,block_errors,total_bits,total_blocks
```

Пример строки:
```
0.00100000,0.00004250,0.00570000,34,114,800000,20000
```

## Теоретические формулы (BSC, hard-decision, код Хэмминга t=1)

Пусть `n = 2^r - 1`, `p` — вероятность ошибки бита в BSC.

BER на выходе декодера:
```
P_b = p - p(1 - p)^(n - 1)
```

BLER (ошибка блока, т.е. >=2 ошибок в кодовом слове):
```
P_bler = 1 - (1 - p)^n - n p (1 - p)^(n - 1)
```
